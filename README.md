1.项目名称
写作盲盒 —— 一个让你愿意动笔的随机命题工具
英文名：writing-blindbox

2.背景与痛点
你有没有过这样的时刻：
想写点什么，打开空白文档，盯着闪烁的光标，脑子也跟着一片空白。
你知道自己想写，但“写什么”成了最大的障碍。

市面上现有的工具：
写作蛙、Perchance等随机生成器：只有抽题，写完之后没地方存，写了就丢了
简书、豆瓣等写作社区：社交压力大，写完没人看很失落，而且题目得自己想
Notion、备忘录等笔记工具：能存，但没有“题目”这个起点，也没有分享功能

3.目标用户
核心用户：
• 想开始写作但不知道写什么的人
• 有写作习惯但偶尔卡壳的人
• 喜欢用文字记录生活、表达自己的人
扩展用户：
• 写作课老师/工作坊带领者（可用来给学生出题）
• 内容创作者需要日常练笔
• 任何想“随便写点什么”的人

4.核心价值
把“选择”变成“拆盒”，把“写作”变成“游戏”。
• 降低启动门槛：不让你面对空白文档，也不让你面对无数个选项犯选择困难症。每次只给一个题目，想写就写，不想写就换。
• 保留选择权：你可以一直摇，直到摇到一个你想写的。没有压力，没有“必须写这个”的绑架。
• 让写作有归处：写下来的东西不会消失。你的个人主页会记录你写过的每一篇，可以回头看，可以看到自己成长的痕迹。
• 可分享，但不强迫：你可以选择把自己的写作分享到“广场”，让别人看到；也可以选择只给自己看。没有点赞、没有评论、没有社交压力。

5.功能列表
v1 单机盲盒
抽题、再摇一次
一个主题（日常盲盒）

v2 能写的盲盒
增加写作区
保存到localStorage
简单历史记录

v3 有身份的盲盒
用户登录
历史记录绑定用户

v4 有朋友的盲盒
广场功能
发布/浏览分享

v5 多主题盲盒
主题切换
主题对应题目池

V6 日常的盲盒
支持导出个人全部写作记录（TXT/Markdown）
每日邮件推送一个题目（可选）
写作统计（连续写了多少天、写了多少字）

6.用户操作流程
1.打开网页，看到一个盲盒（或骰子），点击 “拆开”。
2.页面弹出一个写作题目，例如：“描述你灵魂的形状”。
3.觉得有意思？直接在下方写作区开始写。
4.写完后点击保存，内容存入 “我的历史”。
5.如果愿意，勾选 “发布到广场”，这篇写作会被其他人看到。
6.不喜欢这个题目？点击 “再摇一次”，换一个新题目。

整个过程没有配置、没有教程、没有学习成本。

7.技术实现
部分	选型	说明
前端	HTML/CSS/JS	原生，暂不引入框架
数据存储	先做localStorage，后期接后端	先跑通单机版
后端	Node.js + Express	可选
数据库	MongoDB 或 PostgreSQL	可选
